# Contributor Onboarding Draft

Stateguard is being built as a monorepo with clear architectural boundaries so contributors can work on runtime integrations, adapters, docs, web console, installer flows, or release engineering without stepping on one another.

## Good first areas

- adapter improvements
- Compose detection refinements
- Kubernetes beta manifest modeling
- dashboard polish
- docs and setup guides
- installer and packaging work

## Contribution rule

If a code change alters user-visible behavior, the repo docs should move in the same change set.
